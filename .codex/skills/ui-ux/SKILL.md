# UI/UX Skill

## When To Use

Use this skill for interface design, layout decisions, visual polish, dashboards, tables, forms, navigation, breadcrumbs, modals, skeleton loading, dark mode compatibility, and usability improvements.

## Best Practices

- Design for real workflows and repeated use.
- Keep operational SaaS screens dense, calm, and scannable.
- Use consistent spacing, typography, border radius, colors, and shadows.
- Establish clear hierarchy through size, weight, alignment, and grouping.
- Use whitespace to separate concerns, not to make sparse placeholder layouts.
- Make dashboards actionable with meaningful summaries and drill-down paths.
- Make tables sortable, filterable, paginated, and readable.
- Make forms clear with labels, helper text, validation messages, and logical grouping.
- Use breadcrumbs or contextual headers for deep workflows.
- Support dark mode when surrounding screens support it.
- Use skeletons for content loading when layout is predictable.
- Keep modals focused, constrained, and easy to dismiss safely.

## Things To Avoid

- Do not create marketing-style landing pages for internal tools.
- Do not use decorative elements that compete with workflow content.
- Do not put cards inside cards.
- Do not rely on color alone to communicate status.
- Do not allow text overlap, clipped labels, or unstable layouts.
- Do not add visible instructional text where familiar controls would be clearer.

## Common Mistakes

- Overusing large hero typography inside compact tools.
- Making table actions hard to find.
- Missing empty states that explain what happened and what action is available.
- Using inconsistent status colors across pages.
- Building modals that overflow on mobile.
- Forgetting disabled and permission-limited states.

## Practical Checklist

- Page purpose is obvious in the first viewport.
- Primary actions are visually clear.
- Secondary actions do not compete with primary actions.
- Layout works at mobile, tablet, and desktop widths.
- Forms are labeled and validation is visible.
- Tables support scanning and comparison.
- Empty, loading, error, and success states are polished.
- Dark mode colors remain legible.

## Production Recommendations

- Prefer existing design tokens and component patterns.
- Verify responsive behavior before completing work.
- Check long names, large numbers, and empty data.
- Keep interactive targets large enough for touch.
- Validate contrast for text, icons, borders, and status badges.
